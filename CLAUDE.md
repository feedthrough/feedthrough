# Feedthrough ‚ÄĒ Claude Code instructions

## What this is

A browser debug bridge exposed as an MCP server. `@feedthrough/core` injects into any web page
and intercepts console output, fetch, and XHR. `@feedthrough/mcp` runs locally and exposes
everything as MCP tools over stdio so Claude Code can inspect and drive the page in real time.

The key differentiator: embedded inside the page (not an external Puppeteer/CDP observer), so
it works in any browser and has access to framework internals.

## Common commands


## TypeScript config

- Browser packages (`core`): `module: ESNext`, `moduleResolution: Bundler`, `lib: [DOM, ES2020]`
- Node packages (`mcp`, `cypress`, `vite`, `webpack`): `module: NodeNext`,
  `moduleResolution: NodeNext` ‚ÄĒ relative imports need `.js` extensions
- All packages extend `tsconfig.base.json` at the repo root

## Build quirks

**esbuild CLI is broken in pnpm** ‚ÄĒ the pnpm cmd-shim wrapper calls `node <native-binary>` which
fails. Always use the esbuild JS API instead. See `packages/core/scripts/build-iife.mjs` for the
pattern.

**Cypress adapter has a generated file** ‚ÄĒ `packages/cypress/src/generated/bundle.ts` is
auto-generated by `scripts/embed-bundle.mjs` during `prebuild`. It embeds the core IIFE as a
string constant. It is gitignored; never edit it manually or commit it.

**Build order matters** ‚ÄĒ `@feedthrough/cypress` depends on `@feedthrough/core`'s IIFE output,
so core must be fully built before cypress. `pnpm build` (root) handles this automatically via
workspace dependency ordering.

## Package details

### @feedthrough/core

- `src/transport.ts` ‚ÄĒ WebSocket client with reconnect and outbound queue
- `src/interceptors/console.ts` ‚ÄĒ patches all five console levels, keeps a log buffer
- `src/interceptors/network.ts` ‚ÄĒ wraps `window.fetch` and `XMLHttpRequest`
- `src/commands.ts` ‚ÄĒ dispatches incoming commands: click, fill, hover, inspect, query_dom, etc.
- `src/bridge.ts` ‚ÄĒ `FeedthroughBridge` class, wires everything together
- `src/browser.ts` ‚ÄĒ auto-init entry point for the IIFE bundle; reads config from
  `window.__feedthroughOptions`
- `dist/feedthrough.iife.js` ‚ÄĒ self-contained 9 KB bundle for direct page injection

Public API: `init(options?)` (convenience) and `FeedthroughBridge` class.

### @feedthrough/mcp

- `src/bridge-client.ts` ‚ÄĒ WebSocket server on `:8765`, manages browser connection,
  maps command IDs to pending promises with 10 s timeouts
- `src/server.ts` ‚ÄĒ `McpServer` with nine tools registered via `registerTool` (not the
  deprecated `tool()`). Uses MCP SDK 1.29.0 / Zod v4.
- All logging goes to `process.stderr` ‚ÄĒ stdout is reserved for the MCP stdio protocol
- Port override: `FEEDTHROUGH_PORT` env var

### @feedthrough/cypress

- `src/support.ts` ‚ÄĒ `setupFeedthrough(options?)` hooks `window:before:load` and calls
  `win.eval(bridgeBundle)` to run the bridge in the AUT window scope
- User setup: `import { setupFeedthrough } from "@feedthrough/cypress"` in
  `cypress/support/e2e.ts`, then call `setupFeedthrough()`

### website (Astro)

- `src/pages/index.astro` ‚ÄĒ pure component assembly, no markup or styles of its own
- `src/components/` ‚ÄĒ one `.astro` file per section, scoped styles
- `src/styles/global.css` ‚ÄĒ CSS variables, reset, shared utilities
- `src/layouts/Layout.astro` ‚ÄĒ HTML shell with fonts, meta, scroll observer
- Deploy: GitHub Actions workflow (`.github/workflows/deploy-website.yml`), manual trigger only
  until repo goes public ‚ÄĒ then flip to `push: branches: [main]`

### examples/react-app

Contains three deliberate bugs for testing the MCP diagnostic workflow:
1. Counter increments state by 2 but logs +1 ‚ÄĒ detectable via `click` + `get_console_logs` +
   `inspect_element`
2. Team search is case-sensitive (missing `.toLowerCase()`) ‚ÄĒ detectable via `fill` + `query_dom`
3. Activity feed fetches `/api/events` (wrong URL, 404), error swallowed silently ‚ÄĒ detectable
   via `get_network_requests` + `get_console_logs`

## MCP tools (all nine)

`click`, `fill`, `hover`, `inspect_element`, `query_dom`, `get_console_logs`,
`get_network_requests`, `screenshot` (placeholder), `connection_status`
